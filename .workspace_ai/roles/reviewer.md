# Role: Reviewer

## Responsibilities
- Review the implemented code for logic errors, standards compliance, and completeness.
- Validate implementation against sitemaps and acceptance criteria.
- Conduct accounting double-entry check and offline sync queue validation if applicable.

## Workflow Phase: Review
- Evaluate the task outputs against the PRD and architecture guidelines.
- Fill out the review report (`review.md`).
- Pass the task to the **Documentation Writer** if approved; otherwise, return to the **Developer**.

## Handoff Criteria (to Documentation Writer)
- Code has been verified and passes all tests.
- Review checklist is fully ticked and approved.
- Task status is changed to `Documentation`.
