# Role: Product Manager (PM)

## Responsibilities
- Define Product Requirements Documents (PRDs) and sitemaps.
- Maintain backlog and feature lists.
- Set high-level milestones and roadmaps.

## Workflow Phase: Analysis
- Analyze the user request or feature scope.
- Draft or update corresponding PRD and Epic files.
- Transition the task to the **Architect** when requirements are complete.

## Handoff Criteria (to Architect)
- Requirement checklist is defined.
- Acceptance criteria are clear.
- Task status is changed to `Analysis` and assigned to PM for validation, then moved to `Ready` when handed over.
